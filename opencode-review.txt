[SEC-1] HIGH — XSS via unsanitized innerHTML in track-record/index.html
─────────────────────────────────────────────────────────────────────
File: track-record/index.html, lines 431-440

  tr.innerHTML = `
    <td><button ... onclick="toggleRow('${p.id}')">▶</button></td>
    ...
    <td><strong>${forecastLabel}</strong></td>
    <td><span class="badge ${dirClass}">${p.direction}</span></td>
    ...
  `;

Problems:
  a) p.id is injected into an onclick attribute without escaping.
     A malicious p.id like: x'); alert(document.cookie); //
     would break out of the attribute and execute arbitrary JS.

  b) forecastLabel (p.regime_label or p.signal_name) is rendered
     as raw HTML. A value like <img src=x onerror=alert(1)> would
     execute on page load.

  c) dirClass (p.direction) is used in a CSS class name without
     escaping. A value containing " could break out of the class
     attribute.

  d) p.direction is rendered directly into innerHTML text.

Fix: Use textContent for text values, or pass all dynamic values
through the esc() function that already exists in the codebase.

[SEC-2] HIGH — XSS via JSON.stringify in pre tag
─────────────────────────────────────────────────────────────────────
File: track-record/index.html, line 454

JSON.stringify does NOT escape </pre> or </script> sequences.

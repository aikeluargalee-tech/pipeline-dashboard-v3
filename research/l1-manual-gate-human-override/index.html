The L-1 Manual Gate: Human Override for Geopolitical Risk

Risk Frameworks · 6 min read

Algorithms can detect patterns, but they can't read the news. When a war breaks out, a major exchange gets hacked, or a regulator announces a crackdown, no automated system will catch it fast enough. The L-1 Manual Macro Gate is the human override that sits above all automation.

How It Works

The L-1 gate is set manually using set_gate.py. When activated, it overrides the entire Gate0 system — no matter what the algorithms say, the synthesis verdict becomes "L-1 PAUSE" or "L-1 ABORT — MANUAL OVERRIDE."

Setting the Gate

python3 set_gate.py PAUSE "G7 Iran MOU contested, Hormuz deadline" --expires "2026-06-19T00:00" --trigger "Hormuz confirmed open + oil < $70"

This sets a PAUSE gate with:

  • Reason: Human-readable explanation of the event
  • Expiry: When the gate should auto-deactivate
  • Re-evaluate trigger: The condition that, if met, means it's safe to resume trading

Auto-Expiry

The gate doesn't require manual clearing. When the expiry time passes, collect.py's load_manual_override() function automatically deactivates the gate and logs the auto-expiry. This prevents forgotten gates from blocking trading indefinitely.

When to Use It

Use L-1 for events algorithms can't detect:
  • Geopolitical escalation (wars, sanctions, trade embargoes)
  • Exchange hacks before they're publicly confirmed
  • Regulatory crackdowns (SEC enforcement, country bans)
  • Black swan events not covered by the Black Swan score

Dashboard Display

When the L-1 gate is active, the dashboard shows a prominent red banner at the top with the gate status, reason, and re-evaluation trigger. The synthesis verdict is overridden to reflect the manual gate.

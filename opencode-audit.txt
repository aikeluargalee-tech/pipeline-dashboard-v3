OpenCode Audit — Transient 404 Resolution
=========================================
Date: 2026-07-04
Auditor: OpenCode CLI v1.17.13 (google/gemini-2.5-flash)
Repo: aikeluargalee-tech/pipeline-dashboard-v3

ROOT CAUSE
----------
Transient 404s caused by:
1. Failed deploy (Run #331) served stale cached pages from CDN
2. ~65/70 index.html files had NO cache-control headers
3. Resource references (styles.css, nav.js) had no version query params on most pages
4. Many canonical/OG URLs pointed to wrong repo (missing -v3 suffix)
5. Broken nav.html reference in trap-monitor/index.html

FIXES APPLIED
-------------
[NOSCRIPT] Add cache-control meta tags to ALL index.html (65+ files)
  - <meta http-equiv="Cache-Control" content="no-cache, no-store, must-revalidate">
  - <meta http-equiv="Pragma" content="no-cache">
  - <meta http-equiv="Expires" content="0">

[NOSCRIPT] Bump all styles.css?v=11 → styles.css?v=12 (4 files: root, dashboard, trap-monitor, packet)
[NOSCRIPT] Add ?v=12 to all unversioned styles.css refs (60+ files)
[NOSCRIPT] Bump all nav.js → nav.js?v=12 across all index.html files
[NOSCRIPT] Fix trap-monitor nav.html → nav.js (broken reference, nav.html doesn't exist)
[NOSCRIPT] Remove duplicate styles.css from events-and-disruptions/index.html (referenced twice)
[NOSCRIPT] Bulk fix all pipeline-dashboard/ → pipeline-dashboard-v3/ URLs across all HTML files
          - canonical URLs (root, about, contact, faq, terms, privacy, methodology, glossary, track-record, events-and-disruptions)
          - og:url, og:image, twitter:image
          - schema.org WebSite.url, Organization.url
